# 页面业务理解

## 业务目标

这是一个**地铁/隧道施工地表沉降监测系统**。利用卫星 InSAR（合成孔径雷达干涉测量）技术，监测地面形变速度。工程人员通过这个页面判断哪些区域沉降严重、需要干预。

## 用户操作流程

1. 打开页面 → 看到地图上密密麻麻的彩色点（每个点是一个监测点）
2. 顶栏可以切换数据集、按风险等级筛选（全部/危险/预警/正常）
3. 点击地图上的点 → 右侧弹出详情面板，显示该点的速度、风险等级、时序变化图、施工建议
4. 右侧边栏始终显示风险概览：危险/预警/正常的统计数字、饼图、速度分布直方图、高风险点列表
5. 点击高风险点列表中的条目 → 地图自动飞到该点并打开详情
6. 可以点"巡检"按钮 → 自动逐个巡视 Top 10 高风险点
7. 设置抽屉里可以调整：显示模式（速度/关键日期/阈值分级）、阈值参数、视窗裁剪、危险区聚类

## 核心状态（约 30+ 个 useState）

主要分几组：

**数据状态：**
- `data`（GeoJSON 点集）、`meta`（元信息）、`series`（时序数据）、`zonesData`（危险区）

**选中状态：**
- `selected`/`selectedPoint`（当前选中的点）、`selectedZone`（选中的危险区）

**UI 控制：**
- `mode`（原生/iframe）、`indicator`（显示模式）、`showSettings`、`riskFilter`

**参数配置：**
- `thresholds`（阈值）、`useBbox`（视窗裁剪）、`showZones`、`zoneEpsM`、`zoneMinPts`

**计算派生（useMemo）：**
- `riskSummary`、`riskStats`、`maxAbs`

## 状态联动关系

这是页面最复杂的部分，核心联动链：

```
dataset 变化 → 触发 fetchPoints → data 更新 → riskSummary 重算 → 右侧栏更新
                                              → maxAbs 重算 → 地图重新着色
                                              → 脉冲动画更新（高风险点闪烁）

selectedPoint 变化 → 触发 fetchSeries → panelSeries 更新 → 详情面板时序图更新
                                                          → 施工建议根据风险等级生成

thresholds 变化 → riskSummary 重算 → 所有风险统计联动更新
                → 地图点颜色重新渲染（阈值模式下）
                → localStorage 持久化

showZones 开启 → 触发 fetchZones → zonesData 更新 → 地图叠加危险区多边形
```

## 后端接口交互

| 接口 | 用途 | 触发时机 |
|------|------|----------|
| `GET /insar/datasets` | 获取可用数据集列表 | 页面初始化 |
| `GET /insar/fields?dataset=X` | 获取数据集的字段信息 | dataset 变化 |
| `GET /insar/points?dataset=X&field=Y&bbox=Z` | 获取监测点 GeoJSON | dataset/field/bbox 变化 |
| `GET /insar/series?dataset=X&id=Y` | 获取单点时序数据 | 选中点变化 |
| `GET /insar/zones?dataset=X&...` | 获取 DBSCAN 聚类危险区 | showZones 开启 |
| `GET /static/data/tunnel/YGL_KML.kml` | 加载隧道线路 KML | 地图初始化 |
| `GET /static/data/optical/layers.json` | 加载光学影像图层配置 | 地图初始化 |

## 理解难点

1. **地图渲染逻辑**（L.geoJSON + pointToLayer + onEachFeature）— Leaflet API 不熟会看不懂
2. **里程桩统计（chainage）**— 把点投影到隧道线路上计算里程，涉及向量投影数学
3. **脉冲动画**— setInterval 驱动 CircleMarker 半径和透明度变化
4. **bbox 防抖**— moveend 事件 + setTimeout 200ms + bboxToken 触发重新请求
5. **多层 useEffect 的依赖关系**— 哪个 effect 触发哪个请求，需要理清
