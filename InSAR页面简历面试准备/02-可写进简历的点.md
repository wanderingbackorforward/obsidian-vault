## 点1：基于 Leaflet 的 GeoJSON 大规模点位可视化与交互

**代码证据：**
- `Insar.tsx:799-856`，`pointToLayer` 渲染 CircleMarker，`onEachFeature` 绑定 popup 和 click 事件
- `Insar.tsx:189` `layerByIdRef` 维护 id→layer 映射实现定位
- `Insar.tsx:568` `preferCanvas: true` 开启 Canvas 渲染

**为什么值得写：**
地图可视化是 GIS 类项目的核心能力，5 万点渲染 + 点击交互 + 飞行定位是实打实的功能

**面试官可能追问：**
- 5 万点渲染性能怎么优化的？→ 答：用了 `preferCanvas: true`（Canvas 渲染代替 SVG）、P95 截断色标范围避免极端值影响
- 为什么用 CircleMarker 而不是 Marker？→ 答：CircleMarker 是矢量图形，Canvas 模式下性能好；Marker 是 DOM 元素，几万个会卡死

**需要补懂：**
Leaflet 的 `L.geoJSON`、`pointToLayer`、`CircleMarker` 基本用法；Canvas vs SVG 渲染差异

**风险等级：** 🟢 低

## 点2：多维度风险分级与统计计算

**代码证据：**
- `riskSummary`（useMemo, `Insar.tsx:257-288`）遍历所有点按阈值分级统计
- `riskStats`（`Insar.tsx:290-413`）计算速度直方图 + 正负分布 + 里程桩统计
- `classifyVelocity`（`lib/insar.ts:22-30`）五级分类函数

**为什么值得写：**
展示了前端做数据处理和统计计算的能力，不是简单展示后端返回的数据

**面试官可能追问：**
- 为什么在前端做统计而不是后端？→ 答：阈值是用户可调的，每次调阈值都重新请求后端太慢，前端 useMemo 缓存计算结果，阈值变了立即重算
- useMemo 的依赖数组是什么？→ 答：`[data, thresholds]`，只有数据或阈值变了才重新计算

**需要补懂：**
useMemo 的作用和依赖数组；百分位数计算（percentile 函数）

**风险等级：** 🟢 低

## 点3：点位详情面板 + 时序数据联动

**代码证据：**
- `selectedPoint` 状态变化 → `useEffect`(`Insar.tsx:1259-1290`) 请求 `/insar/series`
- `panelSeriesOption`(useMemo, `Insar.tsx:1293-1306`) 生成 ECharts 配置
- 详情面板渲染时序折线图 + 施工建议（`Insar.tsx:1726-1924`）
- AbortController 取消请求（`Insar.tsx:1265, 1289`）

**为什么值得写：**
典型的"选中→请求→展示"联动模式，面试中很常见的问题

**面试官可能追问：**
- 快速切换点位时怎么避免旧请求覆盖新数据？→ 答：用 `AbortController` 取消上一次请求，`mounted` 标志防止组件卸载后 setState
- useEffect 的 cleanup 函数什么时候执行？→ 答：依赖变化时先执行上一次的 cleanup，再执行新的 effect

**需要补懂：**
AbortController 的用法；useEffect 的 cleanup 函数

**风险等级：** 🟢 低

## 点4：自动巡检功能

**代码证据：**
- `startPatrol`(`Insar.tsx:1525-1561`)，async/await + AbortController
- delay 函数封装（`Insar.tsx:1536-1538`），监听 abort 事件 reject Promise
- 进度条 UI（`Insar.tsx:1703-1724`）

**为什么值得写：**
这是一个有业务价值的交互功能，不是纯 CRUD

**面试官可能追问：**
- 怎么实现的定时？为什么不用 setInterval？→ 答：用 async 循环 + Promise 封装的 delay，配合 AbortController 可以随时中断，比 setInterval 更可控
- 怎么中断的？→ 答：abort 事件触发 delay 的 reject，for 循环 catch 到 AbortError 后退出

**需要补懂：**
async/await 控制流；AbortController 中断异步操作

**风险等级：** 🟢 低

## 点5：动态底图切换 + 瓦片加载容错

**代码证据：**
- `baseLayerName` 状态 → `useEffect`(`Insar.tsx:686-750`) 切换底图
- 监听 `tileerror` 事件，连续 3 次失败在 2 秒内 → 自动回退到上一个可用底图
- `lastGoodBaseLayerNameRef` 记录最后一个成功加载的底图名

**为什么值得写：**
体现了对异常场景的处理意识

**面试官可能追问：**
- 为什么要做自动回退？→ 答：用户可能配置了本地瓦片服务器但没启动，连续加载失败会导致地图空白，自动回退保证可用性
- 为什么是 3 次 2 秒？→ 答：单次 tileerror 可能是网络抖动，连续多次才说明底图源有问题

**需要补懂：**
Leaflet 的 tileerror 事件；错误计数 + 时间窗口的防抖思路

**风险等级：** 🟡 中 — 需要理解 Leaflet 事件机制

## 点6：脉冲动画标记高风险点

**代码证据：**
- `useEffect`(`Insar.tsx:964-1007`)，用 `setInterval(60ms)` 驱动 CircleMarker 的 radius 和 opacity 周期性变化
- `pulseMarkersRef`、`pulsePhaseRef` 管理动画状态

**为什么值得写：**
视觉效果好，面试时可以展示

**面试官可能追问：**
- 为什么用 setInterval 而不是 requestAnimationFrame？→ 答：60ms 间隔约 16fps 够用了，不需要 60fps 的流畅度，且 Leaflet CircleMarker 的 setRadius/setStyle 本身有开销
- 动画怎么清理的？→ 答：useEffect 的 cleanup 里 clearInterval

**需要补懂：**
setInterval 的清理；动画参数（相位、振幅）

**风险等级：** 🟡 中
