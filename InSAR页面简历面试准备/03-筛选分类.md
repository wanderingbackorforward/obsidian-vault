# 筛选分类

## 最推荐写进简历的（3-5 个）

1. **基于 Leaflet 的大规模 GeoJSON 点位可视化**（点1）
   - 核心功能，代码证据充分，容易讲清楚
2. **多维度风险分级与前端统计计算**（点2）
   - 展示数据处理能力，useMemo 优化是加分项
3. **点位详情联动 + 时序数据请求**（点3）
   - 最经典的前端交互模式，AbortController 是亮点
4. **自动巡检功能**（点4）
   - 有业务特色，async/await 控制流值得讲
5. **底图切换容错**（点5）
   - 体现异常处理意识，可以简化表述

## 可以备选但要补看的（3 个）

1. **脉冲动画**（点6）
   - 需要理解 setInterval + Leaflet 动画原理
2. **用户配置持久化**（点8）
   - 太简单，单独写不够分量，可以合并到其他点里
3. **ECharts 图表封装**（EChartsWrapper）
   - 需要理解懒加载 + ResizeObserver

## 明确不建议写的

1. **里程桩统计（Chainage）**（点7）
   - 原因：数学公式复杂，面试官追问向量投影你可能答不上来，风险太高
2. **KML 解析**（kml.ts）
   - 原因：这是工具函数，不是你的核心工作，且 DOMParser 解析 XML 不算前端亮点
3. **上下文提取给 AI 助手**（insarExtractor.ts）
   - 原因：这个功能偏"AI 辅助"方向，和前端开发关系不大，面试官可能觉得奇怪
