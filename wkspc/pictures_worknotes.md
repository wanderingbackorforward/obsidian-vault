# 结构统一标准（所有图必须满足）

每张图的结构必须按以下格式记录：

1. 输入节点（明确输入是什么）

2. 主流程（Step1、Step2、Step3...）

3. 判断节点（必须写成：
   条件 → 分支结果）

4. 并行结构（必须展开为：
   分支A、分支B、分支C）

5. 合并节点（明确汇合点）

6. 输出节点（明确输出结果）

7. 所有节点必须是“可绘制节点”，不得使用概括性描述

不满足以上结构的记录，禁止用于生成提示词

  - 用户类型：管理员、工程师、访客
  - 角色权限：admin（全部功能）、user（核心业务）、guest（数据浏览）
  - 访问控制流程：登录认证→全局认证状态→AuthGuard路由守卫→通过/拒绝

      - 模块配置
      - 数据删除
      - 数据采集
      - 预测分析
      - 工单处理
      - 数据浏览
    - [角色B] user（现场工程师）→ 映射到4项权限：
      - 数据采集
      - 预测分析
      - 工单处理
      - 数据浏览
    - [角色C] guest（访客）→ 映射到1项权限：
      - 数据浏览
