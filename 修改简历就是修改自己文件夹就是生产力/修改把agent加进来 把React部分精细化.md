## Python FastAPI MySQL 线程池 多文档解析 LLM

**公司：华为技术有限公司（实习）**

**项目背景：**
参与华为内部文档质量扫描系统的开发与优化。该系统主要面向内部技术文档、产品文档等多类型文档进行自动化质量检查，整体以 pipeline 流程为主体，结合 FastAPI、MySQL、线程池、多文档解析与 LLM 二次校验能力，实现对标点、语法、链接有效性、内容合规、安全泄露、不规范用语等问题的识别与反馈。

**核心工作与成果：**

### 1. 文档扫描任务流与多文档解析流程

**任务流程：** 参与文档扫描任务的后端流程开发，基于 FastAPI 接收扫描请求，将任务放入线程池异步执行，并通过 MySQL 中的 task 表和文档任务表维护任务状态、文档处理进度及检查结果，支持前端实时查询扫描进度。

**文档解析：** 参与多类型文档的解析与 chunk 化流程，将不同格式文档统一转换为可供 LLM 检查的 chunk 结构。针对不同文档类型设计不同切分策略，例如 PPT 按文本框提取，PDF 则基于文本块位置、缩进、行距等信息进行行合并与段落拼接。

**检查流程：** 参与 common check + 二次筛选的检查链路设计。第一轮通过 LLM 对 chunk 进行高召回检测，识别疑似问题；第二轮结合问题正文、问题类型、修改建议和上下文证据进行二次筛选，仅将证据较明确的问题进入最终反馈环节，降低误报对用户体验的影响。

---

### 2. 多文档链接提取与 PDF 链接对齐优化

**链接提取：** 负责文档 chunk 化过程中的多类型文档链接提取工作，支持在文档解析阶段保留链接信息，为后续链接有效性验证和上下文匹配提供基础数据。

**PDF 难点处理：** 针对 PDF 中文本和链接难以精确对齐的问题进行优化。由于 PDF 本身不是结构化格式，链接通常基于页面坐标区域存储，而文本内容分散在多个文本块中，难以直接建立字级别绑定关系。

**工程方案：** 采用段落级绑定的折中方案，先根据上下位置合并文本行，再基于缩进和行距进行段落拼接；随后将同一页中距离段落最近的 link 绑定到对应段落，最后再进行跨页 chunk 拼接。该方案避免了跨页拼接后位置信息丢失的问题，提高了 PDF 链接上下文提取的稳定性。

---

### 3. 不规范用语与安全合规问题的二次校验

**不规范用语处理：** 针对内部术语容易被 LLM 误判为不规范用语的问题，参与设计从 prompt 白名单到证据打分机制的优化方案。早期方案通过 prompt 编码白名单，并结合规则引擎过滤明显误报，同时支持用户手动维护白名单。

**误报优化：** 针对新术语无法提前加入白名单的问题，设计“证据构建 + 打分机制 + 灰色区域 LLM 二次校验”的处理思路。对疑似不规范用语，统计其在同文档、跨文档中的出现频率，并结合是否出现在标题、定义句、解释性上下文中进行打分，辅助判断其是错误用语还是疑似新术语。

**安全合规检查：** 参与疑似账号密码泄露问题的 prompt 设计，通过上下文区分示例代码、占位符和明显泄露场景。例如占位符可通过，明显真实 token 或密码直接报错，示例代码则根据风险程度进行警告提示。

---

### 4. URL 链接验证与类 ReAct 工具循环设计

**复杂场景分析：** 参与链接有效性验证流程设计，针对 URL 检查中不能仅依赖 HTTP 状态码的问题进行处理。实际场景中可能出现 403 反爬、软 404、首页兜底、登录页跳转、资源迁移、JS 渲染空壳、下载链接返回 HTML 错误页等复杂情况。

**方案设计：** 没有通过大量 if-else 写死所有异常分支，而是将 URL 验证抽象为有限 failure mode，包括明确不可达、访问受限、中间页跳转、资源迁移页、JS 渲染不足、资源类型不匹配、上下文不匹配等。

**类 ReAct 流程：** 参与设计类 ReAct 的链接验证循环，由 LLM 根据 observation 选择下一步工具，包括重试请求、跟随跳转、探测浏览器请求头、抽取页面结构化信息、链接语义匹配和结果标记等。最终将链接结果归类为“存在 / 不存在 / 模糊”，避免在证据不足时强行误判。

**成本控制：** 对链接验证流程进行分流设计，简单问题通过规则或一次性判断处理，复杂链接才进入 LLM 工具循环，减少不必要的模型调用。

## Python FastAPI MySQL 线程池 多文档解析 LLM

**项目背景：**
参与华为内部文档质量扫描系统开发，系统基于 FastAPI、MySQL、线程池、多文档解析与 LLM 二次校验能力，对内部技术文档中的标点、语法、链接有效性、内容合规、安全泄露、不规范用语等问题进行自动化检查。

**核心工作与成果：**

1. **文档扫描 Pipeline 开发：** 参与 FastAPI 任务接入、线程池异步执行、MySQL task 表和文档任务表状态维护等流程开发，支持前端查询文档扫描进度与结果；参与 common check + 二次筛选链路设计，先通过 LLM 高召回识别疑似问题，再结合上下文证据过滤误报。
