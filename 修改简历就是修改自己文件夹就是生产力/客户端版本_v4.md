- **公司**：华为技术有限公司（实习）

- **项目背景**：
    参与华为内部文档质量扫描系统开发。系统面向内部技术文档、产品文档等多类型文档，基于 FastAPI、MySQL、线程池、多文档解析与 LLM 二次校验能力，对标点、语法、链接有效性、内容合规、安全泄露、不规范用语等问题进行自动化检查，并将扫描进度和结果反馈给前端页面。

- **核心工作与成果**：
  * **1. 文档扫描任务流开发**：
    * **技术实现**：参与后端扫描 pipeline 开发，基于 FastAPI 接收扫描任务，通过线程池异步执行文档解析和检查流程，并使用 MySQL 中的 task 表和文档任务表维护任务状态、文档处理进度和问题结果，支持前端实时查询扫描进度。
  * **2. 多类型文档解析与 chunk 化**：
    * **技术实现**：参与多文档解析流程，将不同格式文档统一转换为 LLM 可处理的 chunk 结构。针对不同文档类型设计不同切分策略，例如 PPT 按文本框提取，PDF 基于文本块位置、缩进、行距等信息进行行合并和段落拼接。
  * **3. PDF 链接提取与上下文对齐优化**：
    * **技术实现**：负责多类型文档链接提取工作。针对 PDF 中文本和链接难以精确对齐的问题，采用段落级绑定方案：先根据页面位置合并文本行和段落，再将同页最近链接绑定到对应段落，最后进行跨页 chunk 拼接，避免跨页处理后位置信息丢失，提高 PDF 链接上下文提取稳定性。
  * **4. LLM 二次校验与误报过滤**：
    * **技术实现**：参与 common check + 二次筛选链路设计，第一轮通过 LLM 高召回识别疑似问题，第二轮结合问题正文、问题类型、修改建议和上下文证据进行筛选，减少“原文正确但建议无中生有”的误报。针对不规范用语场景，参与设计白名单、规则过滤和“证据构建 + 打分机制 + LLM 二次校验”方案，通过词频、跨文档分布、标题出现、定义句出现等证据判断疑似新术语或错误用语。
  * **5. URL 验证类 ReAct 工具循环设计**：
    * **技术实现**：参与复杂链接验证流程设计。针对软 404、403 反爬、登录页跳转、首页兜底、资源迁移、JS 渲染不足、资源类型不匹配等问题，将 URL 验证抽象为有限 failure mode，并设计类 ReAct 工具循环，由 LLM 根据 observation 选择重试请求、跟随跳转、请求头探测、页面结构化信息抽取、语义匹配等工具，最终输出“存在 / 不存在 / 模糊”，避免证据不足时强行误判。
