# 1. Pytha3：你可以这样理解

这个是三维设计软件 Web 端。

你不要一上来想 WebGL 多难。先把它降级理解：

> 页面中间是一个三维画布，旁边有 React 写的侧边栏。用户在侧边栏操作图层、材质、视角，最终会影响三维画布里的图元显示。

核心对象就几个：

```text
图元：画布里的一个个 3D 对象，比如墙、板、模型、线条
图层：一组图元的分类管理
材质：贴到模型表面的纹理，比如木纹、石材、颜色
视图：前视图、后视图、左视图、右视图、俯视图、仰视图
UCS：用户坐标系，可以简单理解成当前画布里的坐标参考
```

你从 React 角度理解，就是：

```text
用户点击图层显示/隐藏
        ↓
React 更新图层状态
        ↓
调用接口保存状态
        ↓
通知 WebGL 画布把对应图元显示/隐藏
```

所以“图层显隐状态失效”不神秘，本质就是：

> 前端状态、后端保存的数据、WebGL 画布里的真实显示状态，三者有一个没同步上。

比如可能是：

```text
React 侧边栏显示：图层 A 已隐藏
后端数据库保存：图层 A hidden = true
WebGL 画布实际：图层 A 的图元还显示着
```

那这就是状态不同步。

## 材质库怎么讲

材质库也不用想复杂。

你可以理解成：

> 后台维护一堆材质图片，前端展示材质列表和预览图，用户选择某个材质后，把它应用到三维图元上。

## 孤儿图元怎么讲

这个名字听起来复杂，其实很好理解。

```text
正常情况：
图元 A 属于 图层 1
图元 B 属于 图层 2

异常情况：
图元 C 存在了，但它没有正确关联到任何图层/对象
```

这就是“孤儿图元”。

它的本质是：

> 数据保存到一半失败了，导致某些图元对象存在，但它和图层、模型、工程之间的关系没保存完整。

比如网络异常：

```text
1. 前端创建图元
2. 后端保存图元基本信息成功
3. 但保存图元和图层的关联关系失败
4. 页面再加载时，这个图元不知道属于谁
```

面试这么说就行：

> 孤儿图元本质是网络异常或接口流程中断导致的数据一致性问题。图元本身可能创建成功了，但它和图层、工程或父对象之间的关联关系没有正确保存，所以后续加载时就出现了无法归类或无法正常管理的图元。我当时主要是协助排查保存流程和补充异常处理。

这个说法很稳。

## @Modifying 是什么

你不需要讲太深，只要知道：

在 Spring Data JPA 里，如果 DAO 方法不是普通查询，而是更新或删除，就需要加 `@Modifying`。

比如：

```java
@Modifying
@Query("update User u set u.name = ?1 where u.id = ?2")
int updateNameById(String name, Long id);
```

所以“自动生成 @Modifying”意思是：

> 用户在配置里声明某个 DAO 方法是更新/删除操作时，代码生成器要自动在生成代码里加上这个注解。

面试你说：

> 这块主要是代码模板层面的修改，根据 DAO 方法类型判断是否生成 @Modifying 注解，避免生成出来的更新/删除方法运行时报错。

## docx 上传解析预览怎么理解

这个和你华为文档项目还能接上。

基本流程：

```text
前端选择 docx 文件
        ↓
上传到后端
        ↓
后端解析 docx 内容
        ↓
返回文本 / HTML / 结构化内容
        ↓
前端展示预览
```

React 角度：

```text
<input type="file" />
FormData 上传
接口返回解析结果
setState 保存内容
页面渲染预览区域
```

后端角度：

```text
MultipartFile 接收文件
用 docx 解析库读取内容
转成前端可展示的数据
返回给前端
```

面试说：

> 我做过 docx 上传解析预览，前端主要处理文件选择、上传状态和预览展示，后端负责接收 MultipartFile，解析文档内容后返回给前端。

# 3. Monix：IoT 平台就是普通业务系统

Monix 不要想复杂。

它就是：

> 一堆设备接入平台，然后前端大屏展示设备数量、状态、告警点位。

React 角度：

```text
请求设备列表
请求统计接口
请求告警点位
把数据展示到大屏、表格、地图上
```

后端角度：

```text
查设备表
查告警表
按状态 group by 统计
返回给前端
```

比如“大屏对象数量统计”就是：

```sql
select status, count(*) 
from device 
group by status;
```

或者：

```text
在线设备数量
离线设备数量
告警设备数量
总设备数量
```

GIS 地图红点提示也不用想复杂：

```text
后端返回设备经纬度、告警类型、告警等级
前端在地图上根据经纬度渲染 marker
如果是告警设备，就显示红点
```

面试说：

> Monix 这块更偏业务系统，我主要做设备统计、大屏接口、工单管理和 GIS 地图告警点展示。核心就是后端提供聚合查询，前端根据设备状态和经纬度在地图上分类展示。
