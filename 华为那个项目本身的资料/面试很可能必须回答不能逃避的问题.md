1
这不就是检查 URL 吗？为什么不用普通 HTTP 状态码？
原因是很多状态码本身就是有迷惑性的
例如403可能是整个不完整header 200可能是登录页 可能是首页 

2
那这个不就是ifelse状态机吗 403重试 200就语意匹配

状态机的前提是要要能够完全枚举清楚路径 但是实际上外部网页返回形式是开放的 例如200可能会存在多种情况 例如迁移页 js空壳 首页 等 我这苦lm不负责具体的状态穷举 就让llm负责自己知道错误类型 并且自己判断下一步继续访问或者提取语义或者直接来进行判断

3
那 LLM 的价值到底是什么？是不是最后做一次判断就行？

llm在最后能够确定内容后则可以不需要继续执行tools 但是在这个之前就不行 例如页面200但是让你访问新的链接等

4
你的 Agent 工具有哪些？别只说概念。
1访问页面 支持header和get
2抽取页面工具
3页面匹配工具，语义判断工具

5
Prompt 怎么写？是不是也要穷举各种网页情况？
不是 是选择失败的类型：
例如html空 例如反爬 waf 例如200但是页面不存在 例如跳转中间页
例如语言错误等
