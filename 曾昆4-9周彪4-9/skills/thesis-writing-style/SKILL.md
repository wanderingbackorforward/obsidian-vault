# 学术论文写作风格技能 (Thesis Writing Style Skill)

## 概述

本技能基于范文分析，指导生成符合中文工科学术论文写作规范的高质量内容。重点关注段落结构、语言风格、论证逻辑和学术表达，而非排版格式（排版由 `thesis-formatting` 技能负责）。

参考文件位于 `references/` 文件夹：
- `references/范文.docx` — 原始范文文档
- `references/范文.md` — 范文的文本版本

---

## 触发场景

- "写论文的引言部分"
- "用学术语言描述..."
- "按范文风格改写..."
- "生成文献综述"
- "写一段关于XX技术的研究现状"

## 范文写作特点分析

### 一、整体结构模式

范文采用典型的工科科研报告/专著章节结构：

```
引言
  ├── (1) 研究背景与意义
  ├── (2) 研究内容与目标
  ├── (3) 研究方法与技术路线
  ├── (4) 中期研究任务与考核指标
  └── (5) 中期研究任务完成情况

正文章节（每章）
  ├── 章导言段（概述本章工作和成果）
  ├── X.1 一级节
  │   ├── X.1.1 二级节
  │   └── X.1.2 二级节
  └── X.N 本章小结

文献综述章节
  ├── 技术发展现状（按子方向分节）
  ├── 专利布局现状
  └── 每节末尾：综述性总结段

结论
  ├── 研究进展（分项列举）
  ├── 任务完成情况
  ├── 创新点
  └── 下阶段计划
```

### 二、段落结构特征

#### 1. 段落长度
- 正文段落通常 **150-300字**，信息密度高
- 引言/背景段可达 **300-500字** 的长段落，一气呵成阐述背景
- 技术描述段落 **100-200字**，简洁精确

#### 2. 段落开头模式（主题句位置：段首）

范文中段落几乎全部采用 **段首主题句** 模式：

| 类型 | 开头句式示例 |
|------|------------|
| 背景引入 | "城市大型地下基础设施施工过程中..." |
| 技术定义 | "知识图谱（Knowledge Graph，简称KG）以结构化的形式描述..." |
| 方法说明 | "课题研制的新型自运算智能传感器围绕'多参量感知、边缘计算、快速诊断'构建..." |
| 成果概述 | "2024年度及2025年度，课题在多源数据分类与融合...取得了阶段性成果。" |
| 综述过渡 | "在多源异构数据融合的研究中，Dimitrova等..." |

#### 3. 段落内部展开方式
- **总-分结构**：先给出总论点，再分条展开（大量使用"（1）（2）（3）"编号）
- **递进式**：从基础概念→具体方法→应用场景，层层深入
- **并列式**：多个研究者/方法并列介绍，用"类似的，""此外，""与此同时，"连接

### 三、语言风格特征

#### 1. 学术用语规范

**常用动词搭配：**
- 提出/提出了一种...方法/模型/框架/理念
- 构建/建立/搭建了...模型/体系/架构/平台
- 实现了...的高效/精准/实时...
- 开展了...方面的研究/探索
- 取得了阶段性成果/进展
- 显著提升/有效提高了...的效率/精度/能力

**常用形容词/副词：**
- 高效、精准、实时、动态、系统化
- 显著、有效、初步、进一步
- 多源、异构、多维度、跨平台

**常用连接词/过渡语：**
- 然而/但是（转折）
- 此外/同时/与此同时（并列补充）
- 因此/从而/进而（因果递进）
- 综上所述（总结）
- 为了有效应对这一挑战（目的引导）
- 在此基础上（递进）

#### 2. 句式结构

**典型句式模板：**

```
【背景+问题句】
"在...过程中，...具有...特征，...之间的关系更加复杂。"

【方法+目的句】
"通过...技术/方法，实现了...，从而提升了...。"

【成果+意义句】
"...不仅...，还...，为...提供了...。"

【文献引用句】
"XXX等[引用]提出/研发/构建了一种...，通过...实现了...。"
"类似的，XXX等[引用]提出...方法，...。"

【综述总结句】
"综上所述，当前在...领域，已有诸多研究成果。然而，现有研究仍存在...不足。"

【章节导言句】
"20XX年度，课题在...方面取得了阶段性成果。针对...，系统梳理了...，完成了...。"
```

#### 3. 人称与语态
- **无主语/被动式为主**："采用...方法""通过...实现""可以...""能够..."
- **课题/本文作主语**："课题研制了...""本课题旨在..."
- **绝不使用第一人称**（"我""我们"）

#### 4. 术语处理
- 首次出现时给出中英文全称：`知识图谱（Knowledge Graph，简称KG）`
- 后续使用简称或中文名
- 英文缩写保持原样：InSAR、LiDAR、GPR、BIM、GIS

### 四、文献综述写作模式

范文的文献综述遵循清晰的三段式结构：

```
第一部分：领域概述（1-2段）
  → 阐述该方向的重要性和挑战

第二部分：具体研究梳理（3-5段）
  → 按研究方向/方法分组
  → 每组内按时间或逻辑顺序排列
  → 每篇文献用1-3句话概括：作者+方法+成果

第三部分：总结与不足（1段）
  → "综上所述，...已有诸多成果。然而，...仍存在不足。"
  → 指出研究空白，引出本课题的必要性
```

**文献引用格式：**
- 上标引用：`[1]`、`[2]`
- 引用句式：`XXX等[N]提出/开发/构建了...`
- 多文献并列：按逻辑分组，不简单堆砌

### 五、技术描述写作模式

#### 1. 原理/概念描述
```
定义句 → 组成/分类 → 工作原理 → 优势/特点
```

#### 2. 算法/模型描述
```
算法背景 → 核心公式（带编号） → 变量说明（"式中..."） → 计算步骤 → 应用说明
```

公式描述的固定格式：
- "...表达式如式（X-X）所示："
- "式中：$X$表示...，$Y$表示..."
- "通过计算式（X-X）~式（X-X），可得到..."

#### 3. 系统/平台描述
```
设计理念 → 整体架构 → 各模块功能 → 关键技术 → 应用效果
```

#### 4. 实验/工程描述
```
工程背景 → 技术方案 → 实施过程 → 数据/结果 → 结论
```

### 六、图表引用规范

- 图引用：`如图X-X所示` 或 `见图X-X`
- 表引用：`如表X-X所示` 或 `各参数如表X-X`
- 公式引用：`如式（X-X）所示` 或 `由式（X-X）可得`
- 图表必须在正文中先引用再出现

### 七、章节过渡技巧

**章首导言段**（每章开头必有）：
- 概述本章研究内容和主要成果
- 承接上一章，说明本章的定位
- 示例："2024年度及2025年度，课题在...方面取得了阶段性成果。针对...，系统梳理了...，完成了...的构建，实现了...。"

**章末小结**（每章结尾必有）：
- 总结本章主要工作和成果
- 为下一章做铺垫

**节间过渡**：
- 使用"在此基础上""进一步""同时"等连接词自然过渡

## 写作指南

### 写引言时
1. 先写研究背景与意义（一段长文，300-500字）
2. 再写研究内容与目标（分专题列举）
3. 然后写研究方法与技术路线
4. 最后写任务与指标（如适用）

### 写文献综述时
1. 先概述该方向的重要性（1-2段）
2. 按子方向分组梳理文献（每组3-5篇）
3. 每篇文献：作者+方法+成果（1-3句）
4. 末尾总结不足，引出本课题

### 写技术章节时
1. 章首导言段概述本章工作
2. 先讲原理/概念，再讲具体方法
3. 公式必须有编号和变量说明
4. 图表先引用后出现
5. 章末小结总结成果

### 写结论时
1. 分项列举研究进展
2. 对照任务书说明完成情况
3. 提炼创新点（3-5条）
4. 展望下阶段计划

---

## 注意事项

1. 保持学术写作的 **客观性和严谨性**，避免口语化表达
2. 适应中文学术写作特点：长句为主，逻辑紧密，信息密度高
3. 可灵活应用风格，不必完全照搬，但核心模式应保持一致
4. 术语首次出现必须给出中英文全称
5. 数据和结论必须有依据，避免空泛描述
6. 输出内容应该是高质量的学术文字，可直接用于论文
