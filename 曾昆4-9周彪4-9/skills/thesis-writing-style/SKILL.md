# 学术论文写作风格技能 (Thesis Writing Style Skill)

---
description: 用于按照特定范文风格撰写中文工科学术论文内容。当用户需要撰写论文正文、生成学术性文字、改写润色学术内容，或明确提到"论文写作"、"学术风格"、"按范文写"、"写引言"、"写文献综述"、"写技术章节"、"写结论"时触发。本技能专注于写作风格和语言表达，不涉及排版格式（排版由thesis-formatting技能负责）。务必在用户进行学术写作、论文撰写、文献综述、研究方法描述、技术方案描述、工程案例描述等任务时使用本技能，确保输出符合学术规范和范文风格。
---

## 概述

本技能基于范文分析，指导生成符合中文工科学术论文写作规范的高质量内容。重点关注段落结构、语言风格、论证逻辑和学术表达，而非排版格式（排版由 `thesis-formatting` 技能负责）。

参考文件位于 `references/` 文件夹：
- `references/范文.docx` — 原始范文文档
- `references/范文.md` — 范文的文本版本

- "写论文的引言部分"
- "用学术语言描述..."
- "按范文风格改写..."
- "生成文献综述"

- "写工程案例/示范工程描述"
- "写结论/创新点"
- 用户需要撰写任何学术性文字

## 禁止项（必须严格遵守）

### 语言禁止项

1. **禁止使用第一人称**：不得出现"我""我们""笔者"等第一人称表述
2. **禁止口语化表达**：不得出现"其实""说白了""大概""差不多""挺好的""搞定""弄好"等口语
3. **禁止使用感叹号**：学术写作中不使用"！"
4. **禁止使用反问句**：不得使用"难道不是...吗？""怎么能...呢？"等反问句式
5. **禁止使用修辞手法**：不得使用比喻、拟人、排比等文学修辞
6. **禁止夸张程度副词**：不得使用"非常""特别""极其""超级"等，应使用"显著""有效""初步""进一步"等学术用语
7. **禁止负面评价他人研究**：不得直接说他人研究"差""不好""错误"，应使用"仍存在不足""有待进一步研究""尚未完善"等委婉表述
8. **禁止空泛描述**：不得使用"效果很好""性能不错"等无数据支撑的评价，必须有具体数据或明确依据

### 结构禁止项

1. **禁止段落只有一句话**：每个段落至少包含2-3句话
2. **禁止章节缺少导言段**：每章开头必须有导言段概述本章内容
3. **禁止章节缺少小结**：每章结尾必须有小结总结本章工作
4. **禁止图表先于引用出现**：图表必须在正文中先引用再出现
5. **禁止公式没有编号**：所有公式必须有编号（如"式（2-1）"）
6. **禁止公式没有变量说明**：公式后必须有"式中：..."的变量解释
7. **禁止术语首次出现不给全称**：专业术语首次出现必须给出中英文全称和缩写

### 职责边界

- ✅ 负责：写作风格、语言表达、段落组织、论证逻辑
- ❌ 不负责：排版格式（由thesis-formatting技能处理）
- ❌ 不负责：字体、字号、行距、页边距等格式参数
- ❌ 不负责：页眉页脚、目录生成等文档格式

## 范文写作特点分析

### 一、整体结构模式

范文采用典型的工科科研报告/专著章节结构：

```
引言
  ├── (1) 研究背景与意义（一段长文，500-700字）
  ├── (2) 研究内容与目标（分专题列举）
  ├── (3) 研究方法与技术路线
  ├── (4) 中期研究任务与考核指标（表格形式）
  └── (5) 中期研究任务完成情况（分项列举）

文献综述章节
  ├── X.1 技术发展现状
  │   ├── X.1.1 方向一研究现状
  │   ├── X.1.2 方向二研究现状
  │   └── ...
  ├── X.2 专利布局现状
  └── 每节末尾：综述性总结段（"综上所述，...然而，...仍存在不足"）

正文技术章节（每章）
  ├── 章导言段（概述本章年度成果，200-400字）
  ├── X.1 一级节
  │   ├── X.1.1 二级节
  │   └── X.1.2 二级节
  └── X.N 本章小结（分条列举主要成果）

平台/系统章节
  ├── 需求分析（总体需求→功能需求→性能需求）
  ├── 平台架构设计
  ├── 各功能模块详述
  └── 本章小结

工程示范章节
  ├── 工程背景与概况
  ├── 技术难点分析
  ├── 解决方案与实施
  └── 本章小结

结论
  ├── 研究进展（按类别分项：关键技术、装备、平台、标准、示范工程）
  ├── 任务完成情况（表格对照）
  ├── 创新点（3-5条，每条以"创新点N："开头）
  └── 下阶段计划（分条列举）
```

### 二、段落结构特征

#### 1. 段落长度（按内容类型区分）

| 内容类型 | 典型字数 | 说明 |
|---------|---------|------|
| 引言背景段 | 500-700字 | 一气呵成阐述研究背景，信息密度极高 |
| 章首导言段 | 200-400字 | 概述本章年度工作和阶段性成果 |
| 文献综述段 | 200-500字 | 按研究方向分组梳理，每段覆盖3-6篇文献 |
| 技术原理段 | 150-400字 | 定义→组成→原理→特点 |
| 系统描述段 | 200-500字 | 架构→模块→功能→优势 |
| 本章小结 | 100-300字 | 分条列举，每条1-3句 |

#### 2. 段落开头模式（主题句位置：段首）

范文中段落几乎全部采用 **段首主题句** 模式：

| 类型 | 开头句式示例 |
|------|------------|
| 背景引入 | "城市大型地下基础设施施工过程中..." |
| 技术定义 | "知识图谱（Knowledge Graph，简称KG）以结构化的形式描述..." |
| 方法说明 | "课题研制的新型自运算智能传感器围绕'多参量感知、边缘计算、快速诊断'构建..." |
| 成果概述 | "2024年度及2025年度，课题在多源数据分类与融合...取得了阶段性成果。" |
| 综述过渡 | "在多源异构数据融合的研究中，Dimitrova等..." |
| 需求引出 | "针对超大规模工程建设管理的复杂需求，传统...已显露出明显局限性。" |

#### 3. 段落内部展开方式

- **总-分结构**（最常用）：先给出总论点，再分条展开，大量使用"（1）（2）（3）"编号
- **多级列举结构**：（1）→ 1）→ ①，层层细化，这是范文最显著的组织特征
- **递进式**：从基础概念→具体方法→应用场景，层层深入
- **并列式**：多个研究者/方法并列介绍，用"类似的，""此外，""与此同时，"连接
- **问题-方案式**：先指出不足/挑战，再引出解决方案

#### 4. 编号系统使用规范

范文中大量使用多级编号组织内容，这是最突出的写作特征：

```
第一级：（1）（2）（3）  — 用于章节内主要分类
第二级：1）2）3）        — 用于子项展开
第三级：①②③            — 用于更细的子项（较少使用）
```

使用场景：
- 研究内容分专题列举
- 技术方案的组成部分
- 系统架构的层次描述
- 需求分析的分类
- 结论中的成果列举

### 三、语言风格特征

#### 1. 学术用语规范

**常用动词搭配：**
- 提出/提出了一种...方法/模型/框架/理念
- 构建/建立/搭建了...模型/体系/架构/平台
- 实现了...的高效/精准/实时...
- 开展了...方面的研究/探索
- 取得了阶段性成果/进展
- 显著提升/有效提高了...的效率/精度/能力
- 完成了...的设计/开发/研制
- 初步明确/初步构建了...

**常用形容词/副词：**
- 高效、精准、实时、动态、系统化
- 显著、有效、初步、进一步
- 多源、异构、多维度、跨平台
- 全面、系统、深入、充分

**常用连接词/过渡语：**
- 然而/但是（转折）
- 此外/同时/与此同时（并列补充）
- 因此/从而/进而（因果递进）
- 综上所述（总结）
- 为了有效应对这一挑战（目的引导）
- 在此基础上（递进）
- 针对...问题/需求（问题导向）
- 随着...的快速发展（背景引入）

#### 2. 句式结构

**典型句式模板：**

```
【背景+问题句】
"在...过程中，...具有...特征，...之间的关系更加复杂。"
"随着...的快速发展，...正逐步从...拓展到...。"

【方法+目的句】
"通过...技术/方法，实现了...，从而提升了...。"
"基于...，提出/开发了一种...，实现了...。"

【成果+意义句】
"...不仅...，还...，为...提供了...。"
"...有效提升了...的效率/精度/能力。"

【文献引用句】
"XXX等[引用]提出/研发/构建了一种...，通过...实现了...。"
"类似的，XXX等[引用]提出...方法，...。"

【综述总结句】
"综上所述，当前在...领域，已有诸多研究成果。然而，现有研究仍存在...不足。"
"这些研究不仅涵盖了...等多个方面，还包括...。然而，现有研究仍存在一些不足之处。"

【章节导言句】
"20XX年度及20XX年度，课题在...方面取得了阶段性成果。针对...，系统梳理了...，完成了...。"

【需求引出句】
"针对...的复杂需求，传统...已显露出明显局限性。"
"为应对...的管理复杂性及传统模式局限性，本课题将..."
```

#### 3. 人称与语态

- **无主语/被动式为主**："采用...方法""通过...实现""可以...""能够..."
- **课题/本文/本课题作主语**："课题研制了...""本课题旨在...""本文构建的..."
- **系统/平台/算法作主语**："系统能够...""平台支持...""算法可以..."
- **绝不使用第一人称**（"我""我们""笔者"）

#### 4. 术语处理

- 首次出现时给出中英文全称：`知识图谱（Knowledge Graph，简称KG）`
- 后续使用简称或中文名
- 英文缩写保持原样：InSAR、LiDAR、GPR、BIM、GIS
- 技术名词中的英文保持原文：ESP32-S3、NSGA-Ⅱ、TCN-GRU-Attention

#### 5. 标点符号使用

- 以中文标点为主（句号"。"、逗号"，"、顿号"、"、分号"；"）
- 英文术语内部保持英文标点
- 冒号使用频繁，用于引出列举或说明（"具体功能包括："）
- 书名号用于标准、规范名称
- 不使用感叹号

### 四、文献综述写作模式

范文的文献综述遵循清晰的三段式结构：

```
第一部分：领域概述（1-2段）
  → 阐述该方向的重要性和挑战
  → 指出当前存在的问题

第二部分：具体研究梳理（3-5段）
  → 按研究方向/方法分组
  → 每组内按逻辑顺序排列
  → 每篇文献用1-3句话概括：作者+方法+成果
  → 段落间用"在...的研究中""利用...""针对..."等过渡

第三部分：总结与不足（1段）
  → 固定句式："综上所述，当前在...领域，已有诸多研究成果。"
  → "然而，现有研究仍存在...不足之处。"
  → 指出研究空白，引出本课题的必要性
```

**文献引用格式：**
- 上标引用：`[1]`、`[2]`
- 引用句式：`XXX等[N]提出/开发/构建了...`
- 多文献并列：按逻辑分组，不简单堆砌
- 每段覆盖3-6篇文献，按逻辑关系组织

### 五、技术描述写作模式

#### 1. 原理/概念描述
```
定义句 → 组成/分类 → 工作原理 → 优势/特点
```
示例：先定义知识图谱，再说明其由概念、实体、关系和属性构成，然后分别解释每个组成部分。

#### 2. 算法/模型描述
```
算法背景 → 核心公式（带编号） → 变量说明（"式中..."） → 计算步骤 → 应用说明
```

公式描述的固定格式：
- "...表达式如式（X-X）所示："
- "式中：$X$表示...，$Y$表示..."
- "通过计算式（X-X）~式（X-X），可得到..."

#### 3. 系统/平台描述
```
需求分析（总体需求→功能需求→性能需求）
  → 平台架构设计（分层描述：端层→边缘层→云层）
  → 各功能模块详述
  → 关键技术实现
  → 应用效果
```

#### 4. 传感器/硬件描述
```
设计理念 → 整体架构（分层/分模块）
  → 各模块详述（参数规格、功能说明）
  → 算法/软件部分
  → 性能指标
```

#### 5. 工程案例/示范工程描述
```
工程背景与概况 → 技术难点/挑战
  → 技术方案与解决措施
  → 实施过程与关键数据
  → 成果与效果
```

### 六、图表引用规范

- 图引用：`如图X-X所示` 或 `见图X-X`
- 表引用：`如表X-X所示` 或 `各参数如表X-X` 或 `详见表X-X`
- 公式引用：`如式（X-X）所示` 或 `由式（X-X）可得`
- 图表必须在正文中先引用再出现
- 图表标题格式：`图 X-X 标题内容` / `表 X-X 标题内容`

### 七、章节过渡技巧

**章首导言段**（每章开头必有）：
- 概述本章研究内容和主要成果
- 不需要承接上一章，独立概述即可
- 固定模式："20XX年度及20XX年度，课题在...方面取得了阶段性成果。针对...，系统梳理了...，完成了...的构建，实现了...。"
- 通常200-400字，信息密度高

**章末小结**（每章结尾必有）：
- 分条列举本章主要工作和成果
- 使用"（1）（2）（3）"编号
- 每条1-3句话，简洁概括

**节间过渡**：
- 使用"在此基础上""进一步""同时""针对..."等连接词自然过渡
- 也可通过新节的主题句直接切入，无需刻意过渡

## 写作指南

### 写引言时
1. 先写研究背景与意义（一段长文，500-700字，一气呵成）
2. 再写研究内容与目标（分专题列举，每专题一段）
3. 然后写研究方法与技术路线
4. 最后写任务与指标（如适用，表格形式）

### 写文献综述时
1. 先概述该方向的重要性和挑战（1-2段）
2. 按子方向分组梳理文献（每组3-6篇）
3. 每篇文献：作者+方法+成果（1-3句）
4. 段落间用过渡语连接
5. 末尾固定句式总结不足，引出本课题

### 写技术章节时
1. 章首导言段概述本章年度工作和成果
2. 先讲原理/概念，再讲具体方法
3. 大量使用（1）（2）（3）编号组织内容
4. 公式必须有编号和变量说明
5. 图表先引用后出现
6. 章末小结分条总结成果

### 写系统/平台章节时
1. 先做需求分析（总体→功能→性能）
2. 再描述平台架构（分层描述）
3. 然后详述各功能模块
4. 最后总结平台特点和优势

### 写工程案例时
1. 先介绍工程背景和概况
2. 分析技术难点和挑战
3. 描述技术方案和解决措施
4. 展示实施效果和关键数据

### 写结论时
1. 总起段概述研究背景和本年度工作重点
2. 按类别分项列举研究进展（关键技术、装备、平台、标准、示范工程）
3. 对照任务书说明完成情况（表格形式）
4. 提炼创新点（3-5条，每条以"创新点N："开头）
5. 展望下阶段计划（分条列举）

## 注意事项

### 必须遵守
1. 保持学术写作的 **客观性和严谨性**，避免口语化表达
2. 适应中文学术写作特点：长句为主，逻辑紧密，信息密度高
3. 术语首次出现必须给出中英文全称
4. 数据和结论必须有依据，避免空泛描述
5. 输出内容应该是高质量的学术文字，可直接用于论文
6. 严格遵守所有禁止项

### 灵活性原则
本技能基于范文总结规律，使用时可以：
- 保持核心特征不变（段首主题句、总-分结构、编号系统、学术用语）
- 根据具体内容灵活调整段落长度和结构
- 但必须遵守所有禁止项

### 质量保证
如果输出内容不符合本技能要求，说明使用方式有误，需要重新生成。
